## 📂 Project Naming Conventions

To keep this repository organized, all project directories follow a strict nomenclature:

* **`P[Chapter][Project]` (Book Projects):** Projects sourced directly from the textbook's programming projects. 
    * *Example:* `P21_ticksymbol` $\rightarrow$ Chapter **2**, Programming Project **1** (Tick Symbol).
* **`OP[Chapter][Project]` (Own Projects):** Custom, original projects created to practice concepts from a specific chapter.
    * *Example:* `OP22_fromCdegreestoKandF` $\rightarrow$ Chapter **2**, Own Project **2** (Celsius to Kelvin/Fahrenheit converter).

## 🧠 What is Learned in this Book

K. N. King's textbook provides a comprehensive, modern deep-dive into C. Here is a summary of the core concepts covered and practiced in this repository:

### 1. The Fundamentals (Chapters 1–7)
* **C Basics:** Writing, compiling, and linking simple programs; basic syntax and formatting.
* **Formatted I/O:** Master using `printf` and `scanf` with various conversion specifiers.
* **Control Flow:** Utilizing expressions, selection statements (`if`, `switch`), and loops (`while`, `do-while`, `for`).
* **Basic Types:** Working with integers, floating-point numbers, and characters (`char`), including standard limits and arithmetic conversions.

### 2. Core Structures & Program Organization (Chapters 8–10)
* **Arrays:** Declaring and manipulating one-dimensional and multidimensional arrays.
* **Functions:** Defining functions, passing arguments (pass-by-value), and recursive functions.
* **Program Organization:** Understanding local vs. global variables, scope, block structure, and organizing code into modular structures.

### 3. Pointers, Strings, & Large Programs (Chapters 11–15)
* **Pointers:** Mastering address operators (`&`), dereferencing (`*`), and pass-by-pointer to allow functions to modify arguments.
* **Pointers & Arrays:** Deep-diving into pointer arithmetic and how arrays and pointers closely relate in memory.
* **Strings:** Working with null-terminated character arrays, handling string literals, and using the `<string.h>` library safely.
* **The Preprocessor:** Utilizing directives (`#define`, `#include`, conditional compilation) and macros.
* **Large Programs:** Breaking projects into multiple source files (`.c`), headers (`.h`), and managing builds (e.g., using Makefiles).

### 4. Advanced C & Memory Management (Chapters 16–20)
* **Structures, Unions, & Enumerations:** Creating custom, complex data structures.
* **Dynamic Memory Allocation:** Allocating and freeing memory on the heap using `malloc`, `calloc`, `realloc`, and `free`.
* **Advanced Pointer Usage:** Pointers to pointers, arrays of pointers, and function pointers.
* **Low-Level Programming:** Bitwise operators (`&`, `|`, `^`, `~`, `<<`, `>>`), bit-fields, and interacting with hardware registers.

### 5. The Standard C Library (Chapters 21–27)
* **Input/Output (`<stdio.h>`):** Advanced file handling, stream I/O, and block I/O.
* **Standard Library Utilities:** Utilizing utility libraries (`<stdlib.h>`, `<string.h>`, `<ctype.h>`, `<math.h>`) and handling errors cleanly.
